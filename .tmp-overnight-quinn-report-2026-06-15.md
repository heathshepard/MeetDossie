## Issues Found & Fixed (Quinn → main staging) — 3 commits

### Commit 3a60903 — CORS origin allowlist
**Problem**: `extract-form-fields`, `fill-form`, `draft-amendment` rejected requests from `staging.meetdossie.com` with 403 "Origin not allowed." This blocked EVERY Talk-to-Dossie fill_forms / draft_amendment / initiate_termination flow on staging.

**Fix**: Added `staging.meetdossie.com` to `ALLOWED_ORIGINS` and tightened the Vercel preview regex to `*-heathshepard-6590s-projects.vercel.app` on all 3 endpoints.

### Commit 9c4e8b2 — TREC 24-20 + 25-17 field-key mismatch
**Problem**: `fillNewHomeComplete` (24-20) and `fillFarmRanch` (25-17) were passing keys like `property_address_header`, `total_sales_price`, `cash_down_payment`, `loan_amount`, `earnest_money_amount`, `option_fee_amount` — but the field-map JSONs expect `property_address` / `sales_price` / `cash_portion` / `financing_amount` / `earnest_money` / `option_fee`. Names + dates landed, but property addresses and ALL financial values were silently missing. Forms returned `ok=true` so the bug was invisible at the API layer.

**Fix**: Brought both fillers in line with `fillNewHomeIncomplete` (23-20), which already used the correct keys. Round-2 visual verification confirmed: TREC 24-20 went from 5/11 → 9/9 verified fields, TREC 25-17 went from 4/10 → 10/10.

### Commit 68c7e27 — draft-amendment snake_case + strict-mode safety
**Problem A**: `draft-amendment.js` only accepted `transactionId` / `amendmentType` / `newValue` (camelCase). The React chat dispatcher sends snake_case. Every draft_amendment call from Talk-to-Dossie failed with `transactionId is required.`

**Fix A**: Accept both camelCase (legacy) and snake_case (chat dispatcher) for all three params.

**Problem B**: Strict-mode `fill-form` would silently fill a blank PDF when called with empty `field_values`. Mission spec required loud failure.

**Fix B**: Strict mode now returns 400 `STRICT_MODE_INSUFFICIENT_FIELDS` when fewer than 2 non-empty field values are passed.
