### Root cause — auth middleware call-shape mismatch

`api/_middleware/auth.js` exports `verifySupabaseToken(req)` — it expects the **whole `req` object** so it can read `req.headers.authorization` itself. Working endpoints (e.g. `api/fill-form.js` line 2976) call it as:

```js
const { userId } = await verifySupabaseToken(req);
```

But `api/fill-form-via-docuseal.js` lines 284–291 call it with a raw token string:

```js
const authHeader = req.headers.authorization || '';
const token = authHeader.startsWith('Bearer ') ? authHeader.slice(7) : '';
if (!token) {
  throw new AuthError('Missing authorization token', 401);
}
const decoded = await verifySupabaseToken(token);   // ❌ wrong — passes string, not req
userId = decoded.sub;                                // ❌ wrong — return shape is {userId, email}
```

Inside `verifySupabaseToken`, `extractBearerToken(<string>)` runs `<string>.headers.authorization` which is `undefined` → `AuthError('Missing or malformed Authorization header.')` → 401.

**Fix (two lines):**
```js
const { userId: uid } = await verifySupabaseToken(req);
userId = uid;
```
(Remove the manual `authHeader` / `token` extraction above — middleware does it.)

Even if the call shape were fixed, the next line (`decoded.sub`) would still be wrong — the middleware returns `{ userId, email }`, not Supabase JWT claims directly. So the destructure must be `{ userId }`.
