# Carter loop — fix log_offer offer_price mapping

**Bug:** log_offer dispatcher writes NULL into `transaction_offers.offer_price` for every offer. End-user impact: chat says "Offer logged: **unknown price** from Mary Jane" and the Offers comparison table is unusable. Verified live on staging `cf6ecbf` (preview `2r4i921tk`) — row `358b0f98-0c50-4d0f-b36a-48824c7bfcd7` inserted with `offer_price = NULL` despite the user saying "$260,000".

**Root cause:** parameter name mismatch between tool schema and dispatcher.

**File:** `C:\Users\Heath Shepard\Desktop\Dossie\dossie-app.jsx` line **3070**
```js
offer_price: params.offer_amount,   // params.offer_amount is undefined
```

Tool schema declares the field as `offer_price` (see `MeetDossie\api\chat.js:366`):
```js
offer_price: { type: 'number', description: 'The offer price in dollars' },
```

**Fix:** change line 3070 to `offer_price: params.offer_price,` AND audit lines ~3055-3085 of the log_offer block for any other `params.*` name that doesn't match the schema in `api/chat.js:358-380` (financing_type, down_payment_pct, option_fee, option_days, earnest_money, closing_date, escalation_clause, escalation_cap, notes). Then rebuild bundle, deploy to staging, ping Quinn for re-test of Test F.
