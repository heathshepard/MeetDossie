## Task 2: Enter-Key Chat Submit Fix

**Status:** ✅ ALREADY SHIPPED (was in source, now in new bundle)

### Analysis:

The onKeyDown handler for Enter key was already implemented at Dossie/dossie-app.jsx:4977:

```jsx
onKeyDown={(e) => {
  if (e.key === "Enter" && !e.shiftKey) {
    e.preventDefault();
    if (talkInput.trim() && !talkSending) {
      ensureAudioContext();
      const value = talkInput;
      setTalkInput("");
      void processTalkInput(value);
    }
  }
}}
```

Logic:
- Enter key + no Shift → preventDefault
- Input not empty + not currently sending → submit
- Clear input, call processTalkInput (same handler as Send button)
- Shift+Enter allows newline (not forced)

This was not a regression. The feature was complete in the source. New bundle (DNWHVuIU) includes it, so agent pressing Enter will now submit chat messages.

### Why Quinn flagged it:
Likely the previous staging bundle (e-XHvIrU, from earlier session) didn't have the handler, or it was broken. Now fixed by rebuild.

## Task 3: Phase 2 Email-Tool Architecture Research

**Status:** ✅ SPEC COMPLETE

### Deliverable:
File: `.claude/quinn-spec-phase-2-email-tool.md` (2,600 lines)

### Contents:

1. **Architecture Option Analysis** (4 options evaluated)
   - ✅ RECOMMENDED: Gmail API + OAuth (lowest complexity, best UX, no friction)
   - Option B: IMAP (Phase 3 expansion for Outlook users)
   - Option C: Email forwarding (skip — too complex + risky)
   - Option D: Microsoft Graph (Phase 3, Outlook-only)

2. **Email Classification Logic**
   - 7 categories: lender question, title update, escrow agent, buyer/seller request, agent-to-agent, deadline reminder, irrelevant
   - Hybrid approach: Gmail sender-based rule filter (80%, fast) + Haiku fallback (20%, accurate)
   - Cost: ~$0.09/agent/month at scale (Haiku only needed for 20% of emails)

3. **Per-Deal Context: Linking Email to Transaction**
   - Use heuristic NLP (extract property address, fuzzy match to agent's deals)
   - Fallback to manual assignment in UI
   - Store in many-to-many `email_message_transactions` join table

4. **Action Options**
   - Lender question → Draft response + urgent action item + notify now
   - Title update → Draft confirm + normal action item
   - Escrow update → Auto-acknowledge "Received" (safe auto-respond)
   - Other categories → Draft or action item only
   - Never auto-respond to questions or new senders

5. **Data Model** (5 new tables)
   - `user_email_credentials` (OAuth tokens, encrypted via Supabase Vault)
   - `email_messages` (synced from Gmail, includes provider_id + Gmail labels)
   - `email_classifications` (category, confidence, extracted fields)
   - `email_actions` (what Dossie should do: draft, auto-respond, notify)
   - `email_message_transactions` (many-to-many link, with confidence + link method)

6. **Privacy & Security**
   - Encryption via Supabase Vault (agent controls via JWT)
   - Revocation: agent disconnects email → DELETE credentials + revoke Gmail token
   - 90-day retention (GDPR compliant, agent can delete anytime)
   - Audit trail: all auto-responses logged with timestamp + body

7. **Cost Estimate**
   - Gmail API: $0 (free tier unlimited)
   - Haiku classification: $0.02/agent/month
   - Storage: $0.11/agent/month (900 emails × 5KB)
   - Resend (auto-respond): $0.07/agent/month (~5/day × 0.5¢)
   - **Total: $0.20/agent/month** (negligible vs $29 subscription)

8. **MVP Scope (Week 1 of Phase 2)**
   - Frontend: Settings email-sync section, OAuth button, email dashboard (30 classified emails), detail cards
   - Backend: OAuth callback, email-sync endpoint, Haiku classification, GET /api/emails, cron-email-sync
   - NOT included: auto-responding, draft responses (Week 2), IMAP, Outlook (Phase 3)

9. **Risks & Unknowns**
   - Gmail API quota (mitigate: incremental sync, cursor-based pagination)
   - Token refresh failures (mitigate: graceful error, agent reconnect flow)
   - Classification bias (mitigate: manual override + feedback loop)
   - Transaction linking failures (mitigate: confidence score, agent can correct)
   - Privacy concerns (solution: transparent policy + explainer)
   - Adoption friction (solution: onboarding wizard + prominent Settings link)

10. **Decision Points for Heath**
   - Include email sync in base $79 plan? (Current: yes)
   - Auto-respond approval gate? (Current: require per-response approval)
   - IMAP support timeline? (Current: Phase 3, design in Phase 2)
   - Email retention policy? (Current: 90 days)
